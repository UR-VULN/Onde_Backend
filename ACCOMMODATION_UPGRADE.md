# 🏨 숙소 예약 시스템 고도화 (인원 관리 및 추가 요금 정책)

이 문서는 숙소 예약 시스템에서 누락되었던 **객실 수용 인원 관리** 및 **인원 증가에 따른 추가 요금(Surcharge) 정책** 구현 내용을 정리합니다.

## 🚀 주요 변경 사항

### 1. 데이터베이스 스키마 확장 (`core-module`)
- **마이그레이션 파일 생성:** `V9__Add_Room_Capacity_And_Surcharge.sql`
- **추가된 컬럼 (`rooms` 테이블):**
  - `standard_capacity`: 객실 이용 기준 인원
  - `max_capacity`: 객실 최대 수용 가능 인원
  - `surcharge`: 기준 인원 초과 시 1인당 발생하는 추가 금리 (Decimal)
- **데이터 보정:** 기존 `capacity` 값을 기준으로 새 컬럼들의 초기값을 동기화하여 하위 호환성을 유지했습니다.

### 2. 엔티티 모델 고도화 (`core-module`)
- `Room.java` 엔티티에 새롭게 추가된 DB 컬럼들을 JPA 필드로 매핑했습니다.
- `@Column` 설정을 통해 정밀한 금액 계산을 위한 `precision`, `scale` 옵션을 적용했습니다.

### 3. 숙소 검색 및 조회 로직 개선 (`api-module`)
- **인원 필터링:** 검색 시 요청된 인원(`guests`)이 객실의 `max_capacity`를 초과하는 경우 검색 결과에서 자동으로 제외됩니다.
- **실시간 요금 산출:** 검색 결과의 최저가 표시 시, 단순히 기본가(`base_price`)가 아닌 **(기본가 + 초과 인원 × 추가 요금)**이 반영된 실질 최저가를 계산하여 노출합니다.
- **가용성 체크:** 인원 제한을 만족하면서 해당 기간의 재고(`Inventory`)가 있는 방만 카운트하여 보여줍니다.

### 4. 예약 비즈니스 로직 완성 (`api-module`)
- **엄격한 검증:** 예약 요청 시 최대 인원 초과 여부와 선택한 모든 날짜의 재고 존재 여부를 확인합니다.
- **재고 자동차감:** 예약 성공 시 트랜잭션 내에서 해당 날짜들의 객실 재고를 자동으로 1씩 차감하여 오버부킹을 방지합니다.
- **최종 금액 확정:** (일자별 기본가 + 인원별 추가금)의 총합을 계산하여 `Reservation` 엔티티의 `total_price`로 저장합니다.

### 5. API 엔드포인트 연동 (`api-module`)
- `AccommodationController`의 `reserveRoom` 메서드를 실제 서비스 로직과 연결했습니다.
- **보안 연동:** Spring Security의 `@AuthenticationPrincipal`을 사용하여 현재 로그인한 사용자의 고유 ID를 안전하게 추출하고 예약 정보와 매핑합니다.

## 🛠 적용된 기술 스택
- **Java 17 / Spring Boot 3.2.5**
- **JPA (Hibernate) / QueryDSL**
- **MariaDB / Flyway**
- **Spring Security**

## 💡 기대 효과
- **수익성 개선:** 기준 인원 초과 시 자동으로 추가 요금을 징수할 수 있습니다.
- **운영 안정성:** 객실별 최대 인원 제한을 통해 무분별한 투숙객 증가로 인한 컴플레인을 방지합니다.
- **정확한 정보 제공:** 사용자에게 실제 결제할 금액에 가까운 최저가를 미리 노출함으로써 사용자 경험을 개선합니다.
